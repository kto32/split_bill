# Split Bill API

Небольшой HTTP API на Laravel 12 / PHP 8.2 для разбивки счета между участниками с учетом чаевых и весов.

Главное требование: сумма всех долей всегда точно равна итоговой сумме счета до копейки.

## Требования

- PHP 8.2+
- Composer
- Laravel 12

База данных, Docker, очереди, фронтенд и аутентификация не используются.

## Установка

```bash
composer install
```

Если `.env` отсутствует:

```bash
cp .env.example .env
php artisan key:generate
```

## Запуск

```bash
php artisan serve
```

По умолчанию приложение будет доступно на:

```text
http://127.0.0.1:8000
```

## Endpoint

```http
POST /api/split
Content-Type: application/json
Accept: application/json
```

Пример запроса:

```json
{
  "amount": "100.00",
  "tip_percent": "15",
  "participants": [
    { "name": "Аня", "weight": 1 },
    { "name": "Боря", "weight": 1 },
    { "name": "Вася", "weight": 1 }
  ]
}
```

Пример ответа:

```json
{
  "total": "115.00",
  "shares": [
    { "name": "Аня", "amount": "38.34" },
    { "name": "Боря", "amount": "38.33" },
    { "name": "Вася", "amount": "38.33" }
  ]
}
```

Пример через `curl`:

```bash
curl -X POST http://127.0.0.1:8000/api/split \
  -H 'Content-Type: application/json' \
  -H 'Accept: application/json' \
  --data '{
    "amount": "100.00",
    "tip_percent": "15",
    "participants": [
      {"name": "Аня", "weight": 1},
      {"name": "Боря", "weight": 1},
      {"name": "Вася", "weight": 1}
    ]
  }'
```

## Валидация

Входные данные проверяются в `App\Http\Requests\SplitBillRequest`.

- `amount`: обязательное положительное число, до 2 знаков после точки.
- `tip_percent`: обязательное число от 0 до 100, до 2 знаков после точки.
- `participants`: обязательный массив от 1 до 50 участников.
- `participants.*.name`: обязательная строка.
- `participants.*.weight`: необязательное целое число от 1. Если не передано, используется `1`.

Ошибки валидации возвращаются в едином JSON-формате с HTTP-кодом `422`:

```json
{
  "message": "Validation failed.",
  "errors": {
    "amount": [
      "The amount field must be greater than 0."
    ]
  }
}
```

## Расчет денег

Расчеты не используют `float`.

- Сумма счета парсится в целые копейки: `"100.00"` -> `10000`.
- Процент чаевых парсится в сотые доли процента: `"15"` -> `1500`, `"12.5"` -> `1250`.
- Чаевые считаются целочисленно и округляются до копейки по правилу half-up.
- Итоговая сумма распределяется между участниками пропорционально весам.
- Базовая доля считается через `intdiv`.
- Остаточные копейки раздаются методом наибольшего остатка: сначала участникам с большим остатком от деления, при равенстве - по исходному порядку входного массива.

Такой алгоритм детерминирован: одинаковый вход всегда дает одинаковый ответ.

## Архитектура

- `SplitBillController` - тонкий контроллер: принимает request, вызывает сервис, возвращает JSON.
- `SplitBillRequest` - валидация и преобразование валидных данных в input DTO.
- `SplitBillService` - вся бизнес-логика расчета.
- `DTO/SplitBill/*` - входные и выходные DTO.
- `MoneyFormatter` - преобразование денежных строк в копейки и обратно.
- `PercentFormatter` - преобразование процента в сотые доли процента.

## Тесты

Запуск:

```bash
php artisan test
```

Также можно использовать Composer script:

```bash
composer test
```

Покрыты сценарии:

- ровное деление;
- деление с остатком и проверка, что сумма долей равна `total`;
- чаевые;
- веса участников;
- дробные чаевые и округление half-up;
- крупные суммы;
- максимум 50 участников;
- минимальная сумма `0.01`;
- `tip_percent = 100`;
- отсутствие `weight` и использование значения по умолчанию;
- стабильный tie-break при равных остатках;
- ошибки валидации для невалидной суммы, процента, пустых участников и `weight < 1`.

Подробное объяснение тестов находится в `TESTS_EXPLAINED.md`.
